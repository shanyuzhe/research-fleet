# Findings — internal honest ledger

> One file per experiment/investigation, date-stamped. This folder is where
> the project is **brutally honest**: negative results, kill verdicts, doubts,
> confounds. It is deliberately firewalled from the writer agent — internal
> honesty here, clean narrative there (see CHEATSHEET.md).

## Rules

1. **Negative results are recorded in full** — and every negative finding
   must include a "what this run bought" paragraph (what was ruled out, what
   boundary was learned). Findings are project memory, not a graveyard.
2. Numbers cite files: `path: key=value`. Un-cited numbers get deleted on
   sight.
3. When a finding matures into something the paper needs, it is distilled
   into a claim file (`claims/`) — that distillation is the ONLY road from
   here to the paper.
4. Superseded findings move to `_archive/` with a one-line pointer to what
   replaced them; nothing is deleted.
